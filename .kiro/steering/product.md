# プロダクト定義

## 概要

このドキュメントはLoL Labのプロダクトビジョン、ターゲットユーザー、主要機能を定義します。

## プロダクトビジョン

League of Legendsプレイヤーが戦略的思考を深め、継続的にスキルアップできる包括的な学習プラットフォームを提供する。

## ターゲットユーザー

### プライマリーターゲット
**競技志向のLoLプレイヤー（ランク戦参加者）**
- 年齢層: 16-35歳
- プレイ経験: 1年以上
- 目標: ランク上昇、スキル向上

### ユーザーの課題
1. **情報収集の非効率性**: 対戦相手の情報を毎回手動で調査
2. **戦略知識の蓄積困難**: 学んだ戦略やカウンター情報の記録・管理が困難
3. **分析の属人性**: 個人の記憶に依存した戦略判断

### ユーザーニーズ
1. **効率的な情報収集**: 対戦相手の戦績・傾向を素早く把握
2. **知識の体系化**: 戦略情報の構造化された保存・検索
3. **継続的学習**: 過去の経験を活用した戦略改善
4. **データ駆動判断**: 客観的データに基づく戦略決定

## 主要機能

### 1. サモナー検索・分析
**目的**: 対戦相手の情報を素早く取得・分析

**機能詳細**:
- サモナー名とリージョンで検索
- リアルタイム戦績取得
- ランク情報表示
- 試合履歴分析 (今後)

**ユーザー価値**:
- 対戦前の情報収集時間を短縮
- 相手の実力と調子を客観的に把握

### 2. チャンピオンノート管理
**目的**: マッチアップ別の戦略を記録・蓄積

**機能詳細**:
- マッチアップ別戦略記録
- ルーン・アイテム・スペル設定保存
- 戦略メモ作成・編集
- ノート検索・フィルタリング (今後)

**ユーザー価値**:
- 学んだ戦略の体系的な蓄積
- 過去の経験を素早く参照
- 継続的な戦略改善

### 3. データ永続化
**目的**: ユーザー固有データの安全な保存

**機能詳細**:
- ユーザー認証 (Google OAuth)
- ユーザー固有データ保存
- セッション間データ維持

**ユーザー価値**:
- デバイス間でのデータ同期
- データの安全な保管

## 成功指標

### 定量的指標
- **ユーザー継続率**: 月次アクティブユーザー70%以上
- **機能利用率**: コア機能（検索・ノート作成）の週次利用80%以上
- **レスポンス性能**: API応答時間平均3秒以内
- **データ品質**: 検索成功率95%以上

### 定性的指標
- **ユーザー満足度**: 戦略分析効率の向上実感
- **学習効果**: ランク上昇への貢献実感
- **使いやすさ**: 直感的操作による継続利用

## ユーザーストーリー

### エピック1: サモナー分析
**As a** 競技志向プレイヤー  
**I want to** 対戦相手の戦績と傾向を素早く分析したい  
**So that** 効果的な戦略を立てることができる

#### US-1.1: サモナー検索
- サモナー名とリージョンで検索できる
- 検索結果が5秒以内に表示される
- 無効な入力に対して適切なエラーメッセージが表示される

#### US-1.2: 戦績表示
- 現在のランク情報が表示される
- サモナーレベルが表示される
- プロフィールアイコンが表示される

### エピック2: 戦略ノート管理
**As a** 学習志向プレイヤー  
**I want to** チャンピオン別の戦略を記録・管理したい  
**So that** 知識を蓄積し継続的に改善できる

#### US-2.1: ノート作成
- マッチアップ（自分のチャンピオン vs 相手チャンピオン）を指定できる
- ルーン・アイテム・スペル設定を保存できる
- 戦略メモを記述できる

#### US-2.2: ノート一覧表示
- 作成したノートの一覧が表示される
- チャンピオン画像が表示される
- 作成日時が表示される

#### US-2.3: ノート編集・削除
- 既存のノートを編集できる
- 不要なノートを削除できる

### エピック3: ユーザー体験
**As a** 全てのユーザー  
**I want to** 快適で安全なアプリケーション体験をしたい  
**So that** ストレスなく継続利用できる

#### US-3.1: 認証
- Googleアカウントでログインできる
- ログイン状態が維持される
- 安全にログアウトできる

#### US-3.2: パフォーマンス
- ページが素早く読み込まれる
- API応答が3秒以内に返る
- スムーズな画面遷移

#### US-3.3: ユーザビリティ
- 直感的なUI
- モバイル・デスクトップ対応
- 適切なエラーメッセージ

## 受け入れ条件

### サモナー検索機能
- [ ] 有効なサモナー名とリージョンで検索が成功する
- [ ] 無効な入力に対して適切なエラーメッセージが表示される
- [ ] 検索結果が5秒以内に表示される
- [ ] ランク情報が正確に表示される

### ノート管理機能
- [ ] ノートの作成・編集・削除が正常に動作する
- [ ] ルーン・アイテム・スペル情報が正確に保存される
- [ ] ユーザー固有のデータ分離が保証される
- [ ] データの永続化が確実に行われる

### システム品質
- [ ] API応答時間が平均3秒以内である
- [ ] セキュリティ脆弱性が存在しない
- [ ] モバイルデバイスで正常に動作する

## ビジネス要件

### 技術要件
- **スケーラビリティ**: 同時ユーザー1000人対応
- **可用性**: 99.5%以上のアップタイム目標
- **セキュリティ**: 個人データ保護とAPI安全性確保

### 運用要件
- **メンテナンス性**: モジュラー設計による保守性
- **拡張性**: 新機能追加の容易性

## 用語定義

### ドメイン用語
- **サモナー**: League of Legendsのプレイヤーアカウント
- **チャンピオン**: ゲーム内で操作可能なキャラクター
- **マッチアップ**: 特定のチャンピオン同士の対戦組み合わせ
- **ルーン**: 試合前に設定する能力強化システム
- **スペル**: サモナースペル（Flash, Igniteなど）
- **リージョン**: サーバー地域（NA, KR, JP等）

### 技術用語
- **API**: Application Programming Interface
- **OAuth**: Open Authorization（認証プロトコル）
- **CRUD**: Create, Read, Update, Delete
- **JWT**: JSON Web Token
