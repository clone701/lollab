# エラーハンドリング動作確認チェックリスト

## 概要

このドキュメントは、Supabase Auth移行におけるエラーハンドリングの手動テストチェックリストです。Task 5.3の実施時に使用してください。

**対応要件**: 8.1, 8.2, 8.3, 8.4

---

## テスト環境準備

### 前提条件
- [ ] Supabase Auth移行が完了している（Task 1-4が完了）
- [ ] 開発サーバーが起動している（`npm run dev`）
- [ ] ブラウザの開発者ツールを開いている（コンソールログ確認用）
- [ ] テスト用のGoogleアカウントを2つ以上用意している

### 初期状態の確認
- [ ] ログアウト状態でテストを開始する
- [ ] ブラウザのキャッシュとCookieをクリアする
- [ ] コンソールにエラーが表示されていないことを確認

## 参考情報

### Supabase Authエラーコード
- `invalid_grant`: OAuth認証失敗
- `network_error`: ネットワーク接続エラー
- `PGRST301`: RLSポリシー違反（未認証）
- `PGRST116`: データベース接続エラー

### デバッグ用コマンド

```javascript
// Supabase Clientの状態確認
const { createClient } = await import('./src/lib/supabase/client.ts');
const supabase = createClient();
const { data: { session } } = await supabase.auth.getSession();
console.log('Current session:', session);

// RLSポリシーのテスト
const { data, error } = await supabase.from('champion_notes').select();
console.log('Notes:', data, 'Error:', error);
```
