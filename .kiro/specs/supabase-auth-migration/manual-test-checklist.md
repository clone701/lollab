# 認証フロー手動テストチェックリスト

## 概要

このドキュメントは、Task 5.1「認証フローの動作確認」のための包括的な手動テストチェックリストです。Supabase Auth移行後の認証機能が正しく動作することを確認します。

## テスト環境準備

### 前提条件

- [ ] フロントエンドアプリケーションが起動している（`npm run dev`）
- [ ] Supabase Dashboardで Google OAuth設定が有効化されている
- [ ] 環境変数が正しく設定されている
  - `NEXT_PUBLIC_SUPABASE_URL`
  - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- [ ] ブラウザのキャッシュとCookieをクリアしている（推奨）

### テスト用アカウント

- **Googleアカウント1**: メインテスト用
- **Googleアカウント2**: マルチユーザーテスト用（オプション）

### 推奨ブラウザ

- Chrome（最新版）
- Firefox（最新版）
- Safari（最新版）

---

## テストケース

### 1. ログイン機能の確認

#### 1.1 初回ログイン

**目的**: Google OAuth認証が正常に開始されることを確認

**手順**:
1. ブラウザでアプリケーションのホームページ（`http://localhost:3000`）を開く
2. Navbarに「ログイン」ボタンが表示されていることを確認
3. 「ログイン」ボタンをクリック

**期待される結果**:
- [ ] Google OAuth認証画面にリダイレクトされる
- [ ] Googleアカウント選択画面が表示される
- [ ] エラーメッセージが表示されない

**検証要件**: 要件 1.1, 3.1, 3.2

---

#### 1.2 Google OAuth認証の完了

**目的**: Google認証が成功し、アプリケーションに戻ることを確認

**手順**:
1. Google認証画面でテスト用Googleアカウントを選択
2. 必要に応じて権限の許可を行う
3. 認証完了後、アプリケーションにリダイレクトされるのを待つ

**期待される結果**:
- [ ] アプリケーションのホームページにリダイレクトされる
- [ ] Navbarに「ログイン」ボタンが消える
- [ ] Navbarにユーザーアバター画像が表示される
- [ ] アバター画像がGoogleアカウントのプロフィール画像と一致する
- [ ] ローディング状態が適切に表示される（短時間）

**検証要件**: 要件 1.2, 1.3, 3.3, 3.6

---

#### 1.3 認証後のユーザー情報表示

**目的**: 認証成功後、ユーザー情報が正しく表示されることを確認

**手順**:
1. Navbarのユーザーアバター画像をクリック
2. ドロップダウンメニューが表示されることを確認

**期待される結果**:
- [ ] ドロップダウンメニューが表示される
- [ ] メニューにユーザー名またはメールアドレスが表示される
- [ ] メニューに「ログアウト」ボタンが表示される
- [ ] アバター画像が円形で適切なサイズで表示される

**検証要件**: 要件 3.3, 3.4

---

### 2. セッション永続化の確認

#### 2.1 ページリロード後のセッション維持

**目的**: ページをリロードしてもログイン状態が維持されることを確認

**手順**:
1. ログイン状態でブラウザのリロードボタンをクリック（F5キー）
2. ページが再読み込みされるのを待つ

**期待される結果**:
- [ ] ページリロード後もログイン状態が維持される
- [ ] Navbarにユーザーアバター画像が表示される
- [ ] 「ログイン」ボタンが表示されない
- [ ] ローディング状態が短時間表示される（UIのちらつきが最小限）

**検証要件**: 要件 4.1, 4.2

---

#### 2.2 ブラウザタブを閉じて再度開く

**目的**: ブラウザタブを閉じて再度開いてもセッションが維持されることを確認

**手順**:
1. ログイン状態でブラウザタブを閉じる
2. 新しいタブでアプリケーションのホームページを開く

**期待される結果**:
- [ ] ログイン状態が維持される
- [ ] Navbarにユーザーアバター画像が表示される
- [ ] 再度ログインする必要がない

**検証要件**: 要件 4.1, 4.2

---

#### 2.3 別のページに移動してから戻る

**目的**: ページ遷移後もセッションが維持されることを確認

**手順**:
1. ログイン状態でノートページ（`/notes`）に移動
2. ブラウザの戻るボタンでホームページに戻る

**期待される結果**:
- [ ] ページ遷移後もログイン状態が維持される
- [ ] Navbarにユーザーアバター画像が表示される
- [ ] セッション情報が失われない

**検証要件**: 要件 4.1, 4.4

---

### 3. ログアウト機能の確認

#### 3.1 ログアウトボタンのクリック

**目的**: ログアウトボタンをクリックして未認証状態に戻ることを確認

**手順**:
1. ログイン状態でNavbarのユーザーアバター画像をクリック
2. ドロップダウンメニューから「ログアウト」ボタンをクリック

**期待される結果**:
- [ ] ログアウト処理が実行される
- [ ] Navbarのユーザーアバター画像が消える
- [ ] Navbarに「ログイン」ボタンが表示される
- [ ] ホームページにリダイレクトされる（または現在のページに留まる）

**検証要件**: 要件 1.5, 3.5

---

#### 3.2 ログアウト後のページリロード

**目的**: ログアウト後、ページをリロードしても未認証状態が維持されることを確認

**手順**:
1. ログアウト後、ブラウザのリロードボタンをクリック（F5キー）

**期待される結果**:
- [ ] ページリロード後も未認証状態が維持される
- [ ] Navbarに「ログイン」ボタンが表示される
- [ ] ユーザーアバター画像が表示されない

**検証要件**: 要件 4.3

---

#### 3.3 ログアウト後の再ログイン

**目的**: ログアウト後、再度ログインできることを確認

**手順**:
1. ログアウト状態で「ログイン」ボタンをクリック
2. Google OAuth認証を完了する

**期待される結果**:
- [ ] 再度ログインできる
- [ ] Navbarにユーザーアバター画像が表示される
- [ ] セッションが正常に確立される

**検証要件**: 要件 1.1, 1.2, 1.3

---

### 4. データアクセスの確認（オプション）

#### 4.1 認証済みユーザーのノートアクセス

**目的**: 認証済みユーザーがノートページにアクセスできることを確認

**手順**:
1. ログイン状態でノートページ（`/notes`）に移動

**期待される結果**:
- [ ] ノートページが正常に表示される
- [ ] ノート作成フォームが表示される
- [ ] 既存のノート一覧が表示される（存在する場合）
- [ ] リダイレクトされない

**検証要件**: 要件 5.2

---

#### 4.2 未認証ユーザーのノートアクセス

**目的**: 未認証ユーザーがノートページにアクセスできないことを確認

**手順**:
1. ログアウト状態でノートページ（`/notes`）に直接アクセス（URLバーに`http://localhost:3000/notes`を入力）

**期待される結果**:
- [ ] ノートページにアクセスできない
- [ ] ログインページまたはホームページにリダイレクトされる
- [ ] 適切なメッセージが表示される（オプション）

**検証要件**: 要件 5.3

---

### 5. エラーハンドリングの確認（オプション）

#### 5.1 Google OAuth認証のキャンセル

**目的**: Google OAuth認証をキャンセルした場合の動作を確認

**手順**:
1. 「ログイン」ボタンをクリック
2. Google認証画面で「キャンセル」または「戻る」をクリック

**期待される結果**:
- [ ] アプリケーションに戻る
- [ ] 未認証状態が維持される
- [ ] エラーメッセージが表示される（オプション）
- [ ] アプリケーションがクラッシュしない

**検証要件**: 要件 8.1

---

#### 5.2 ネットワークエラーのシミュレーション

**目的**: ネットワークエラー時の動作を確認

**手順**:
1. ブラウザの開発者ツールを開く（F12キー）
2. Networkタブで「Offline」モードを有効にする
3. 「ログイン」ボタンをクリック

**期待される結果**:
- [ ] 適切なエラーメッセージが表示される
- [ ] アプリケーションがクラッシュしない
- [ ] ユーザーに再試行を促すメッセージが表示される（オプション）

**検証要件**: 要件 8.2

---

### 6. UI/UXの確認

#### 6.1 ローディング状態の表示

**目的**: 認証処理中にローディング状態が適切に表示されることを確認

**手順**:
1. 「ログイン」ボタンをクリック
2. 認証処理中のUIを観察

**期待される結果**:
- [ ] ローディングインジケーターが表示される（スピナー、ローディングテキストなど）
- [ ] ボタンが無効化される（連続クリック防止）
- [ ] UIのちらつきが最小限に抑えられる

**検証要件**: 要件 3.6

---

#### 6.2 レスポンシブデザインの確認

**目的**: モバイルデバイスでも認証機能が正常に動作することを確認

**手順**:
1. ブラウザの開発者ツールでモバイルビューに切り替え
2. ログイン・ログアウトの一連の操作を実行

**期待される結果**:
- [ ] モバイルビューでもログインボタンが表示される
- [ ] ユーザーアバター画像が適切なサイズで表示される
- [ ] ドロップダウンメニューが正常に動作する
- [ ] タッチ操作が正常に動作する

**検証要件**: 要件 3.1, 3.3, 3.4

## トラブルシューティング

### よくある問題と解決方法

#### 問題1: Google OAuth認証画面が表示されない

**原因**:
- Supabase DashboardでGoogle OAuth設定が有効化されていない
- 環境変数が正しく設定されていない

**解決方法**:
1. Supabase Dashboard > Authentication > Providers > Googleを確認
2. `.env.local`ファイルの環境変数を確認
3. アプリケーションを再起動

---

#### 問題2: ログイン後、ユーザー情報が表示されない

**原因**:
- AuthContextが正しく実装されていない
- Supabase Clientのセッション取得に失敗している

**解決方法**:
1. ブラウザのコンソールでエラーメッセージを確認
2. `AuthContext.tsx`の実装を確認
3. Supabase Clientの初期化を確認

---

#### 問題3: ページリロード後、ログイン状態が失われる

**原因**:
- セッション永続化が正しく設定されていない
- ブラウザのlocalStorageが無効化されている

**解決方法**:
1. `createBrowserClient`の設定を確認
2. ブラウザのlocalStorageが有効になっているか確認
3. ブラウザのプライベートモードを使用していないか確認

---

#### 問題4: RLSポリシーエラーが発生する

**原因**:
- データベーススキーマ変更が正しく実行されていない
- RLSポリシーが正しく設定されていない

**解決方法**:
1. Supabase Dashboard > SQL Editorでスキーマを確認
2. `champion_notes`テーブルの外部キー制約を確認
3. RLSポリシーが有効になっているか確認

---

## 追加テスト項目（オプション）

### マルチユーザーテスト

**目的**: 複数のユーザーがそれぞれ独立したデータにアクセスできることを確認

**手順**:
1. ユーザー1でログインし、ノートを作成
2. ログアウト
3. ユーザー2でログインし、ノート一覧を確認
4. ユーザー1のノートが表示されないことを確認

**期待される結果**:
- [ ] ユーザー2はユーザー1のノートを見ることができない
- [ ] RLSポリシーが正しく機能している

---

### セッション期限切れテスト

**目的**: セッションが期限切れになった場合の動作を確認

**手順**:
1. ログイン後、長時間（数時間）放置
2. ページをリロードまたは操作を実行

**期待される結果**:
- [ ] セッション期限切れ時、未認証状態に遷移する
- [ ] 適切なメッセージが表示される（オプション）
- [ ] 再ログインを促される
