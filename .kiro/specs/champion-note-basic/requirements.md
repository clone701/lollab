# 要件定義書: ノートページ基本レイアウト

## はじめに

このドキュメントは、LoL Labアプリケーションのノートページ（`/notes`）の基本レイアウトとタブナビゲーションの要件を定義します。この機能は、ノート管理システムの土台となるUI骨組みを提供します。

本Specでは、以下のみを実装します：
- タブナビゲーション（新規ノート作成・汎用ノート・チャンピオン対策ノート）
- 左サイドバーのチャンピオン選択UI（共通コンポーネント）
- 各タブのプレースホルダー表示

**実際のノート作成・編集・一覧表示機能は別Specで実装します。**

## 用語集

- **Notes_Page**: `/notes` ページ全体
- **Tab_Navigation**: 3つのタブを切り替えるナビゲーション
- **Champion_Selector_Sidebar**: 左サイドバーのチャンピオン選択UI
- **Tab_Content_Area**: 右側のメインコンテンツエリア
- **Placeholder**: タブ選択時に表示される仮のコンテンツ
- **My_Champion**: 自分が使用するチャンピオン
- **Enemy_Champion**: 対戦相手のチャンピオン
- **Favorite_Champions**: よく使うチャンピオンのリスト

## 要件

### 要件1: ノートページの基本レイアウト

**ユーザーストーリー:** プレイヤーとして、ノート機能の入り口となるページにアクセスしたい。そうすることで、ノート管理機能を利用できる。

#### 受け入れ基準

1. ノートページは `/notes` URLでアクセス可能である
2. ノートページは認証済みユーザーのみ機能を利用可能である
3. WHEN 未認証ユーザーがアクセスする場合、ノートページはログインが必要であることを伝えるメッセージを表示する
4. ノートページは2カラムレイアウトを提供する（左サイドバー + 右メインエリア）
5. ノートページはレスポンシブデザインに対応する

### 要件2: タブナビゲーション

**ユーザーストーリー:** プレイヤーとして、ノートの種類を切り替えたい。そうすることで、目的に応じたノート機能にアクセスできる。

#### 受け入れ基準

1. タブナビゲーションは3つのタブを表示する
   - 新規ノート作成
   - 汎用ノート
   - チャンピオン対策ノート
2. タブナビゲーションはデフォルトで「新規ノート作成」タブを選択状態にする
3. WHEN ユーザーがタブをクリックする場合、タブナビゲーションは選択状態を視覚的に表示する
4. タブナビゲーションは選択されたタブに対応するコンテンツを表示する
5. タブナビゲーションはページ上部に配置される

### 要件3: 左サイドバーのチャンピオン選択UI

**ユーザーストーリー:** プレイヤーとして、チャンピオンを視覚的に選択したい。そうすることで、素早く正確にチャンピオンを指定できる。

#### 受け入れ基準

1. チャンピオンセレクターサイドバーは左側に固定幅で表示される
2. チャンピオンセレクターサイドバーは以下のセクションを含む
   - 「自分のチャンピオンを選択」セクション（クリック可能なボックス）
   - 「相手のチャンピオンを選択」セクション（クリック可能なボックス）
   - チャンピオン検索入力欄（見出しなし）
   - 「よく使うチャンピオン」セクション
   - 「チャンピオン一覧」セクション
3. チャンピオンセレクターサイドバーはチャンピオン名で検索できる機能を提供する
4. チャンピオンセレクターサイドバーは選択モード（自分/相手）を視覚的に表示する
5. チャンピオンセレクターサイドバーはスクロール可能である
6. 自分のチャンピオンを選択すると、自動的に相手のチャンピオン選択モードに切り替わる
7. 選択ボックスをクリックすると、そのモードに切り替わり、チャンピオンを変更できる

### 要件4: よく使うチャンピオン表示

**ユーザーストーリー:** プレイヤーとして、よく使うチャンピオンを素早く選択したい。そうすることで、効率的にノートを作成できる。

#### 受け入れ基準

1. よく使うチャンピオンは横スクロール可能なリストとして表示される
2. よく使うチャンピオンはチャンピオン画像を円形で表示する
3. よく使うチャンピオンはチャンピオン名を画像下に表示する
4. よく使うチャンピオンは最大10チャンピオンを表示する
5. WHEN チャンピオンをクリックする場合、よく使うチャンピオンは選択状態を更新する

### 要件5: チャンピオン一覧表示

**ユーザーストーリー:** プレイヤーとして、全チャンピオンから選択したい。そうすることで、任意のチャンピオンでノートを作成できる。

#### 受け入れ基準

1. チャンピオンセレクターサイドバーは全171チャンピオンを一覧表示する
2. チャンピオンセレクターサイドバーはチャンピオン画像、日本名（左）、英名（右・グレー）を表示する
3. チャンピオンセレクターサイドバーはチャンピオン名のアルファベット順で表示する
4. チャンピオンセレクターサイドバーは検索クエリに一致するチャンピオンのみを表示する
5. チャンピオンセレクターサイドバーは現在の選択モードに応じたチャンピオンをハイライト表示する

### 要件6: チャンピオン検索機能

**ユーザーストーリー:** プレイヤーとして、チャンピオン名で検索したい。そうすることで、目的のチャンピオンを素早く見つけられる。

#### 受け入れ基準

1. チャンピオンセレクターサイドバーは検索入力欄を提供する
2. チャンピオンセレクターサイドバーは入力に応じてリアルタイムでフィルタリングする
3. チャンピオンセレクターサイドバーは部分一致検索をサポートする
4. チャンピオンセレクターサイドバーは大文字小文字を区別しない検索をサポートする
5. IF 検索結果が0件の場合、チャンピオンセレクターサイドバーは「該当するチャンピオンが見つかりません」と表示する

### 要件7: 新規ノート作成タブのコンテンツ

**ユーザーストーリー:** プレイヤーとして、新規ノート作成タブを選択したい。そうすることで、新しいノートを作成する準備ができる。

#### 受け入れ基準

1. WHEN 「新規ノート作成」タブが選択されている場合、タブコンテンツエリアは左サイドバーを表示する
2. タブコンテンツエリアは右側にプレースホルダーメッセージを表示する
3. タブコンテンツエリアは「チャンピオンを選択してください」というメッセージを表示する
4. タブコンテンツエリアは「左のパネルで自分のチャンピオンと相手のチャンピオンを選択してください」という説明を表示する

### 要件8: 汎用ノートタブのコンテンツ

**ユーザーストーリー:** プレイヤーとして、汎用ノートタブを選択したい。そうすることで、汎用ノート機能にアクセスできる。

#### 受け入れ基準

1. WHEN 「汎用ノート」タブが選択されている場合、タブコンテンツエリアはプレースホルダーメッセージを表示する
2. タブコンテンツエリアは「汎用ノート機能は別Specで実装予定です」というメッセージを表示する
3. タブコンテンツエリアは左サイドバーを非表示にする

### 要件9: チャンピオン対策ノートタブのコンテンツ

**ユーザーストーリー:** プレイヤーとして、チャンピオン対策ノートタブを選択したい。そうすることで、既存の対策ノートを確認できる。

#### 受け入れ基準

1. WHEN 「チャンピオン対策ノート」タブが選択されている場合、タブコンテンツエリアは左サイドバーを表示する
2. タブコンテンツエリアは右側にプレースホルダーメッセージを表示する
3. タブコンテンツエリアは「チャンピオンを選択してください」というメッセージを表示する
4. タブコンテンツエリアは「対策ノート一覧機能は別Specで実装予定です」という説明を表示する

### 要件10: レスポンシブデザイン

**ユーザーストーリー:** プレイヤーとして、モバイルデバイスでもノートページを快適に使用したい。そうすることで、どのデバイスからでもアクセスできる。

#### 受け入れ基準

1. WHEN 画面幅が768px未満の場合、ノートページは左サイドバーを折りたたむ
2. ノートページはハンバーガーメニューボタンを表示する
3. WHEN ハンバーガーメニューをクリックする場合、ノートページは左サイドバーをオーバーレイ表示する
4. タブナビゲーションはモバイルデバイスで適切に表示される
5. チャンピオンセレクターサイドバーはモバイルデバイスで適切に表示される

### 要件11: チャンピオンデータの管理

**ユーザーストーリー:** 開発者として、チャンピオンの静的データを効率的に管理したい。そうすることで、チャンピオン選択UIを素早く表示できる。

#### 受け入れ基準

1. ノートページはチャンピオンデータを定義するchampions.tsファイルを使用する
2. チャンピオンデータはチャンピオンID、名前、画像パスを含む
3. チャンピオンデータは全171チャンピオンのデータを含む
4. チャンピオンデータはTypeScript型定義を提供する
5. ノートページはチャンピオンデータをメモリ上にキャッシュする

### 要件12: 認証状態の確認

**ユーザーストーリー:** プレイヤーとして、ログインしていない状態でノート機能にアクセスできないようにしたい。そうすることで、自分のデータが保護される。

#### 受け入れ基準

1. WHEN 未認証ユーザーがノートページにアクセスする場合、ノートページはログインが必要であることを伝えるメッセージを表示する
2. ノートページはNextAuth.jsのuseSessionフックを使用して認証状態を確認する
3. WHEN セッションが読み込み中の場合、ノートページはローディング表示をする
4. ノートページは認証済みユーザーのIDを状態管理に使用する

### 要件13: ローディング状態の表示

**ユーザーストーリー:** プレイヤーとして、ページ読み込み中であることを視覚的に確認したい。そうすることで、アプリケーションが応答していることを理解できる。

#### 受け入れ基準

1. WHEN ページが読み込み中の場合、ノートページはローディングインジケーターを表示する
2. ノートページはグローバルローディングコンポーネント（GlobalLoading）を使用する
3. WHEN チャンピオンデータが読み込み中の場合、チャンピオンセレクターサイドバーはローディング表示をする

### 要件14: パフォーマンス

**ユーザーストーリー:** プレイヤーとして、ノートページが素早く表示されてほしい。そうすることで、ストレスなくノート機能を利用できる。

#### 受け入れ基準

1. ノートページはページを2秒以内に表示する
2. チャンピオンセレクターサイドバーはチャンピオン一覧を1秒以内に表示する
3. ノートページはチャンピオン画像を遅延読み込み（lazy loading）する
4. ノートページは不要な再レンダリングを防ぐためにReact.memoを使用する
5. タブナビゲーションはタブ切り替えを即座に反映する

## 技術的制約

- Next.js 15 (App Router)を使用
- TypeScript 5を使用
- React 19を使用
- Tailwind CSS 4を使用
- NextAuth.jsを使用して認証状態を管理
- 既存の共通コンポーネント（Panel、GlobalLoading）を使用

## 非機能要件

- ページ読み込み時間: 2秒以内
- チャンピオン一覧表示時間: 1秒以内
- タブ切り替え: 即座に反映
- モバイルデバイス（画面幅768px未満）でのレスポンシブ対応
- アクセシビリティ: キーボード操作対応、適切なARIAラベル

## 実装範囲外（別Specで実装予定）

以下の機能は本Specの範囲外とし、別Specで実装します：

- **新規ノート作成機能**: チャンピオン選択後の作成フォーム、ルーン・スペル・アイテム選択、保存機能
- **汎用ノート機能**: 汎用ノートのCRUD操作、一覧表示
- **チャンピオン対策ノート一覧・編集機能**: 既存ノートの一覧表示、編集・削除機能

本Specでは、これらの機能のプレースホルダーのみを表示します。

## 依存関係

本Specは以下のSpecに依存します：

- **Spec 1-1 (basic-ui-structure)**: ルートレイアウト、ナビゲーションバー、認証システム
- **Spec 1-2 (common-components)**: Panel、GlobalLoading、スタイル定数
