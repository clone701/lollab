# コアステアリングガイドライン

## 概要

これは開発作業を開始する前に**必ず**最初に読まなければならない主要なステアリング文書です。Kiroプロジェクトにおける多言語ドキュメント管理、ユーザーインタラクション、開発ワークフローの基本ガイドラインを定義します。

## 重要な読み順

**この文書は他のステアリング文書を参照したり開発作業を開始する前に必ず最初に読まなければなりません。**

この文書を読んだ後、以下の順序で進んでください：
1. #[[file:subSteering/ai-coding-workflow.md]] - 開発ワークフローガイダンス
2. 必要に応じて`subSteering/`内の他のステアリング文書
3. 技術仕様のための関連する#[[file:docs/]]ファイル

## 言語使用ルール

### ユーザーインタラクション
- **AIアシスタントの応答**: 常に日本語でユーザーに応答する
- **エラーメッセージ**: ユーザー向けエラーを日本語で表示する
- **UIテキスト**: すべてのユーザーインターフェーステキストは日本語にする

### 技術ドキュメント
- **仕様書**: 英語で最初に作成し、その後日本語翻訳を提供する
- **コードコメント**: コードコメントと技術ドキュメントには英語を使用する
- **APIドキュメント**: 英語で維持し、必要に応じて日本語要約を提供する

## ドキュメント構造

### フォルダ構成
```
.kiro/
├── specs/           # 英語仕様書
├── steering/        # 英語ステアリング文書（最初に読む）
└── steering-ja/     # ステアリング文書の日本語翻訳

ja/
├── specs/           # 仕様書の日本語翻訳
└── steering/        # 日本語ステアリング文書（最初に読む）
```

### 同期要件

1. **主要作成**: 常に英語版を最初に作成する
2. **翻訳タイミング**: 英語版の直後に日本語翻訳を作成する
3. **更新プロセス**: 英語文書を更新する際は、日本語版も同時に更新する
4. **バージョン管理**: 英語版と日本語版のバージョンパリティを維持する

## 実装ガイドライン

### AIアシスタント向け
- 開発作業前にこの`steering.md`ファイルを**必ず**最初に読む
- 初期インタラクションからユーザーの言語設定を検出する
- セッション全体を通じて一貫した言語を維持する
- 技術仕様は英語のまま、ユーザー向けコンテンツは日本語に切り替える
- #[[file:subSteering/ai-coding-workflow.md]]で定義された5段階AIコーディングワークフローに従う

### 開発者向け
- コードと技術ドキュメントは英語で記述する
- 複雑なビジネスロジックには日本語コメントを提供する
- ユーザー向け文字列は日本語で作成し、英語フォールバックを用意する

## 品質標準

### 翻訳品質
- 自然で専門的な日本語を使用する
- 技術的正確性を維持する
- 英語版のフォーマットと構造を保持する
- すべての文書で一貫した用語を使用する

### ドキュメント保守
- 英語文書が変更された際に翻訳をレビューする
- 両バージョンが機能的に同等であることを保証する
- タイムスタンプとバージョン番号を一貫して更新する

## ファイル命名規則

- 英語ファイル: 標準命名（例：`requirements.md`）
- 日本語ファイル: それぞれの`ja/`フォルダ内で同じ命名（例：`requirements.md`）
- 英語版と日本語版で同一のフォルダ構造を維持する

## 必須読み込み要件

### 開発作業開始前：
1. この`steering.md`ファイルを完全に読む**必要がある**
2. ワークフローガイダンスのために#[[file:subSteering/ai-coding-workflow.md]]を読む**必要がある**
3. 技術仕様のために適切な#[[file:docs/]]ファイルを参照する**必要がある**
4. 例外なく5段階開発ワークフローに従う**必要がある**

- 適切なドキュメントを参照し検証する**必要がある**
- 英語版と日本語版の一貫性を維持する**必要がある**
